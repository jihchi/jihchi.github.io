CLI coding agents

AI coding agents are now part of both my professional and personal development workflows.

At work

I use Claude Code with Opus 4.7 as my main driver, and Codex CLI with GPT-5.5 interchangeably depending on the task.

One pattern: use Opus to develop a feature, then GPT-5.5 to review changes—or vice versa. Because my company does not set a hard token limit and encourages AI exploration, I generally raise thinking levels instead of optimizing for token savings.

Personal & OSS

For side projects and OSS, I use the Pi coding agent with GPT-5.5 and GLM 5.1.

I switch to Gemini CLI when I want access to Gemini 3.1 Pro.

The practical setup is not “one perfect model.” It is a small rotation of agents and models, each useful for different development moments.

Sandbox experiments

I’m experimenting with Lima to run Pi in an isolated instance. Lima supports syncing with the working directory, which keeps the workflow ergonomic.

limactl create --mount-none --name=default template:debian
# lima is an alias for "limactl shell default".
lima curl https://mise.run/bash | sh
lima mise u -g node@lts
lima mise u -g ripgrep fd npm:@earendil-works/pi-coding-agent
cd ~/my-project
lima --sync . pi

After exiting Pi, Lima shows an interactive prompt:

⚠️ Accept the changes?  
→ Yes  
  No  
  View the changed contents

What I want from a sandbox

1Mount only needed files
2Install coding agent
3Run in isolation
4Review changes
5Accept or discard

Self-hosted assistants

Family Hermes Agent

I have a Hermes Agent for family matters, connected to a WhatsApp group via a WhatsApp Business account. It uses GPT-5.5 and runs in a container on my dedicated server.

Family members interact with it in WhatsApp. I granted the agent access to a Google Calendar and shared that calendar with family members. Everything else lives in an Obsidian Vault as long-term memory.

Personal playground

I also keep a personal playground for agent experiments: a Hermes Agent for personal LLM Wiki practice and assistance using GLM 5.1.

I run my experimental assistant project, Muxclaw, using the same model. It is connected to a Telegram group via a Telegram bot.

Infrastructure and self-hosted services

I rent VPSes and dedicated servers, and use Tailscale to build a secure private network with public access blocked by default.

Self-hosted services are connected through this private network. Docker is the default option for services on VPSes and dedicated servers. I don’t use Kubernetes here; instead, I use Ansible to manage servers.

Everyday natural language assistant

AI is my language-learning partner and a powerful translation tool.

I use it to proofread English responses and writing. It helps with grammar, tone, natural phrasing, alternative expressions, and more. I often ask for alternative ways to express the same thoughts and verify the meaning by reviewing the options.

Brainstorming, Q&A, and research

I find AI useful as a learning tool. I use it to brainstorm ideas and explore current or unfamiliar topics.

For example, while hunting for a job, I use web search and deep research to learn about a company, understand its products, prepare questions for interviews, or summarize relevant background.

Future-self note: this setup will probably change quickly. The durable pattern is not the exact model list—it is the habit of matching the tool, context, and risk level to the task.
