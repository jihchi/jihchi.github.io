My current AI use cases and setup in 2026

As LLMs evolve quickly, I'd like to record how I currently use AI-powered tools. This is mainly for my future self, not a universal recommendation list, and it will likely be out of date soon.

I've been living in the terminal for many years, so I'm comfortable with CLI tools. Here are some system-level tools I use:

Category Tools
Operating system macOS (laptop) and Linux (PC, Omarchy based on Arch)
Command line shell Fish shell
Editor Helix editor for coding and editing. I used to use Visual Studio Code
Dotfiles manager yadm. Alternative Files is great for managing OS-specific dotfiles
CLI tools, environment vars, tooling mise

CLI coding agents

AI coding agents are now part of both my professional and personal development workflows.

At work, I use Claude Code with the Opus 4.7 model as my main driver, and I use Codex CLI with GPT-5.5 interchangeably depending on the task. For example, I might use Opus 4.7 to develop a new feature and then use GPT-5.5 to review the code changes, or vice versa. Because the company does not set a hard limit on tokens and we are encouraged to explore AI tools, I generally set the thinking level higher and don't worry about reducing token consumption in order to maximize the ROI.

For personal side projects and OSS work, I use the Pi coding agent with the GPT-5.5 and GLM 5.1 models, but I switch to the Gemini CLI for accessing Gemini 3.1 Pro model.

Sandbox

I'm experimenting with Lima to run the Pi coding agent in an isolated instance. Lima supports syncing with the working directory, which is a nice ergonomic feature. Here is how I configure it:

limactl create --mount-none --name=default template:debian
# lima is an alias for "limactl shell default".
lima curl https://mise.run/bash | sh
lima mise u -g node@lts
lima mise u -g ripgrep fd npm:@earendil-works/pi-coding-agent
cd ~/my-project
lima --sync . pi

After exiting pi, you’ll see an interactive prompt:

⚠️ Accept the changes?  
→ Yes  
  No  
  View the changed contents

Besides Lima, I plan to experiment with the following:

  • superradcompany/microsandbox: based on libkrun
  • smol-machines/smolvm: based on libkrun
  • afshinm/zerobox: powered by OpenAI Codex's sandbox runtime
  • lukehinds/nono: OS security primitives. Landlock on Linux, Seatbelt on macOS
  • stanford-scs/jai: an ultra-lightweight jail for AI CLIs on modern Linux

From a usability standpoint, virtual machine- and containerization-based solutions are similar: you mount only the necessary folders or files, install the coding agent, and run it there.

Self-hosted assistants

I have a Hermes Agent for family matters, connected to a WhatsApp group via a WhatsApp Business account. The agent uses the GPT-5.5 model and runs in a container on my dedicated server. Family members interact with it in the WhatsApp group. I've granted the agent access to a Google Calendar and shared that calendar with family members. Everything else is stored in an Obsidian Vault (markdown files collection) as long-term memory.

Besides stable family usage, I also keep a personal playground for agent experiments. I have a Hermes Agent for personal LLM Wiki practice and assistance. This agent uses the GLM 5.1 model. I also run my experimental assistant project, Muxclaw, using the same model. It's connected to a Telegram group via a Telegram bot.

Infrastructure

I rent VPSes and dedicated servers, and I use Tailscale to build a secure private network with public access blocked by default. Self-hosted services are connected through this private network. Containerization with Docker is essentially the default option for running services on these VPSes and dedicated servers. I don't use Kubernetes in this setup; instead, I use Ansible to manage the servers.

Self-hosted services

I've been self-hosting the following services:

  • Immich: Self-hosted photo and video management solution
  • Karakeep: Quickly save links, notes, and images
  • Changedetection: Monitor websites for changes and receive alerts.
  • Databasus: PostgreSQL backup tool

Everyday natural language assistant

AI is my language-learning partner and a powerful translation tool. I use it to proofread my English responses and writing. It helps with grammar, tone, natural phrasing, alternative expressions, and more. I often use it to find alternative ways to express the same thoughts and verify the meaning by reviewing those options.

Brainstorming, Q&A and research

I find it useful to use AI as a learning tool. I use it to brainstorm ideas and explore current or unfamiliar topics.

For example, while hunting for a job, I use web search and "deep research" to learn about a company, understand its products, prepare questions for interviews, or summarize relevant background.
